# 任务角色
你现在是一位舆情信息考证与关键事实/证据提取的顶尖高手。你的任务不是总结舆情报告，也不是输出建议，而是把两份舆情报告中适合用于进行多阶推理和洞察的内容，拆解为标准化证据卡。

# 输入材料
1. 《海底捞舆情合集260609-0708》
2. 《汉堡王舆情合集260609-0708》
3. 《证据卡_海底捞集团2025-2026年5月月度微观动态扫描》——作为格式参考

# 目标
从两份舆情报告中筛出：
- 可作为强证据的内容
- 可作为弱证据的内容
- 应判定为噪声、不得入卡的内容

只抽取“可归因、可定位、对既有判断有增量作用”的事实性内容。
不得把报告作者的建议、总结、情绪化评论、热榜标题直接当作证据卡。

最终输出两份证据卡markdown文件：《证据卡_海底捞舆情合集260609-0708》和《证据卡_汉堡王舆情合集260609-0708》

# 拆卡原则
你必须遵守以下规则：

1. 只保留“事实型信号”
- 有明确事件、时间、主体、内容
- 最好有来源名、平台名、监管通报、媒体报道、投诉平台记录等可定位出处

2. 不要把下列内容做成证据卡
- 建议句
- 结论句
- 情绪化评论
- 缺乏出处的泛化判断
- 单纯“热度高”“讨论很多”但没有业务含义的内容

3. 一条证据卡只表达一个原子事实
- 不要把多个事件揉成一张卡

4. 如果一条内容只是支持背景、但不足以推动判断更新
- 标记为 weak
- 不要强行拔高

5. 如果一条内容明显不适合进入判断系统
- 不生成证据卡
- 记录到“排除清单”中，并说明排除原因

# 输出结构
请输出两个部分：

## Part A：证据卡
使用 YAML 代码块逐条输出，格式尽量参照参考附件3。
字段建议如下：

- id
- card_type: "evidence_card"
- title
- entity
- source_type
- evidence_type
- business_dimension
- affected_customer
- signal_type
- signal_theme
- statement
- source_quote

- geo_scope
- related_brands

另外新增 2 个字段：
- bayes_signal_strength: "strong" / "weak" / "noise"
- update_relevance: 这条证据更可能影响哪些已有判断（如：品牌信任、加盟扩张、经营模型、治理质量、短期客流修复等）

特别说明：
1.字段"id"的编写规则
    - 品牌缩写+"SR"+生成日期YYMMDD+两位流水号（从01开始）
    - 品牌缩写：海底捞="HiHP"，汉堡王="HBGK"
    - id编写举例：HiHPSR26070901
2.字段"business_dimension"统一赋值:"Customer" 

## Part B：排除清单
用表格列出未入卡内容：
- 原句/原段摘要
- 所属客户
- 排除原因
- 排除类型（建议/评论/重复背景/无出处/噪声）

# 质量要求
1. statement 必须是中性事实句，不得带建议口吻
2. source_quote 必须尽量贴近原文
3. 若来源只是舆情报告二次转述，source_grade 不得虚高
4. 不允许把“作者观点”伪装成“证据”
5. 最后给出一个简短统计：
- 强证据数量
- 弱证据数量
- 噪声数量
- 最值得进入后续多阶推理和洞察的前5张卡
