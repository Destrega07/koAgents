## 1. 扩容网络硬超时防线并完善异常捕获 (修复 app.js 内的 `runValuePropPipeline`)
由于价值主张小参谋需要生成极其庞大的方案矩阵与多行 Markdown 决策表格，整体生成和流式传输耗时较长（约 20-30 秒），原先的 25 秒熔断策略极易引发误杀。

```javascript
  const timeoutId = setTimeout(() => controller.abort(), 60000); // 为超长表格生成流提供充足通道
```

- **优雅降级异常捕获**：在 catch (e) 块中，加入对 AbortError 的专门判断。如果是超时引起的，弹出 toast("网络响应稍慢，系统正在努力为您加载并解析数据档案...")，并尝试将当前已收集到的残余增量数据进行局部渲染，避免彻底退化为假死状态。
