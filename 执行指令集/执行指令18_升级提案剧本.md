# 角色与目标
你是一位精通原生 JavaScript 异步组件流渲染、多轮数据参数联邦包装的全栈开发专家。
我们需要升级现有的【提案剧本】（`renderProposal`）模块。在原先基于客户生成 5 步剧本的基础上，新增基于“特定合作伙伴画像”定制讲解话术与现场沟通 Tips 的二次流式联调功能[cite: 50, 51]。

# 详细重构需求说明

## 1. 顶部控制栏双选下拉联动重构 (修改 `renderProposal`)
- **DOM 注入**：重构【提案剧本】模块顶部的控制面板（`renderCustomerContextBar`）[cite: 51]。在现有的 “客户选择” 下拉选框的右侧，并排插入一个 **“合作者选择”** 下拉选框（`<select id="proposalCollabSelect" class="select">`），以及一个高亮的主操作按钮 **`{定制提案话术}`**。
- **数据源动态解构**：每当用户切换 “客户选择” 下拉框或刷新页面时，系统需自动在 `state.collaborators` 集合中执行过滤，找出所有 `customerId` 等于当前客户 ID 的合作伙伴[cite: 51]，并将他们的姓名 (`name`) 作为 `<option>` 动态填充进 “合作者选择” 下拉框中[cite: 51]。若当前客户下无合作者，该下拉框及定制按钮自动设为 `disabled` 变灰色。

## 2. 跨组件二次多轮数据打包与网络联调
- 当销售人员选定某个具体合作伙伴（如 Icy Guo），并点击 `{定制提案话术}` 按钮时，前端执行以下参数拼装[cite: 51]：
  - **数据 A**：通过 `getModuleData(cid, "proposal")` 异步读取当前数据库中已经存在的第一阶段 5 步原生提案剧本纯文本（`stored.content`）[cite: 51]。
  - **数据 B**：从 `state.collaborators` 中抓取该合作伙伴全量的结构化 YAML/字段属性（包含 `personality` 性格、`commStyle` 沟通风格、`decisionPreference` 决策偏好、`commSuggestions` 三条建议数组）[cite: 51]。
  - **请求分发**：将数据 A 与数据 B 统一序列化为一个名为 `collaborator_customization_context` 的 JSON 字符串。
  - **参数包投喂**：作为第一层级参数拼装成 `reqBody`，再次调用流式打字机组件 `streamCozeAnswerText` 发起 Fetch 请求，目标保持 Bot ID（`7553662475095867442`）不变[cite: 51]。

## 3. 实现“不覆盖追加”双容器流式打字机渲染
- **双容器隔离**：请修改 `renderProposal` 报告渲染面板的 DOM 结构[cite: 51]。在原先展现 5 步剧本的容器下方，紧临新建一个带有浅色调背景卡片效果的独立渲染槽容器：`<div id="proposalCollabCustomZone" class="needles-report-section" style="margin-top: 20px; border-left: 4px solid var(--gold);"></div>`[cite: 51]。
- **打字机打靶隔离**：当点击定制按钮并接收到 Coze 的新一轮 Delta 流时，流式监听器 `onDelta` 必须**只定向对齐这个新增的 `#proposalCollabCustomZone` 容器进行 innerHTML 或打字机追加渲染**，绝对不允许刷新或覆写原有上方的基本剧本容器。
- **落库定义**：在第二次流式传输完全正常结束后，将返回的定制话术与 Tips 文本追加存储到 IndexedDB 数据库中该客户 `proposal` 记录的增量新字段 `collaboratorCustomContent` 中，确保离线刷新页面后，双容器内容同时完好重绘呈现[cite: 51]。

## 4. 行动提示与导出系统同步适配
- **同步升级 {导出txt文件} 逻辑**：当销售人员点击导出时，前端需将上方原有的“5步通用剧本文本”与下方生成的“合作伙伴定制讲解话术及Tips”通过 `\n\n===[针对该合作者的专属话术定制]===\n\n` 物理线进行拼接合成后，再触发 TXT 文件下载，确保两部分话术在一张纸上完整归仓。

# 交付标准
1. 点击定制话术按钮后，原有剧本完好保留，下方独立容器精准开启打字机效果，顺畅喷吐出面向 Icy 的现场沟通 Tips。
2. 彻底跑通双文本区块独立编辑保存及合并导出 TXT 的闭环链路。
3. 交付重构后的 `renderProposal` 核心 DOM 追加及定制点击响应函数代码片段。
