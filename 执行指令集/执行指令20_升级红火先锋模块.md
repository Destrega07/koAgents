## 2. 增加多周周三日期生成算法 (修改 app.js)
新增辅助函数，用于根据当前日期自动计算出包含当周在内、倒推近 6 周的所有周三 8 位日期戳（YYYYMMDD）数组：

```javascript
function getRecentWednesdayYmdList(weeksCount = 6) {
  const list = [];
  const now = new Date();
  // 找到最近的一个周三（若今天就是周三则为今天）
  const day = now.getDay();
  const diffToWed = (day >= 3 ? day - 3 : day + 4);
  const latestWed = new Date(now.getFullYear(), now.getMonth(), now.getDate() - diffToWed);

  for (let i = 0; i < weeksCount; i++) {
    const d = new Date(latestWed.getTime() - i * 7 * 24 * 3600 * 1000);
    const yyyy = d.getFullYear();
    const mm = String(d.getMonth() + 1).padStart(2, "0");
    const dd = String(d.getDate()).padStart(2, "0");
    list.push(`${yyyy}${mm}${dd}`);
  }
  return list; // 返回如 ['20260722', '20260715', '20260708', ...]
}
```

## 3. 重构前端自动化直连多周补齐与状态机(重构 renderKaSentiment)
定位到 renderKaSentiment 函数内 {召唤红火哨兵生成报告} 按钮的 onclick 事件，彻底废弃本地 file-picker 控件弹窗，升级为自动向 COS 发起“多周增量探测”：  

1. 状态锁定：点击按钮瞬间，按钮转为 disabled = true 状态，文字闪烁显示：“红火哨兵正在搜寻云端近6周舆情快照...”。  
2. 客户映射与本地排重：
- 获取当前选中的客户名称，从 KA_CUSTOMER_CONFIG 中匹配得出 customerKey 和 filePrefix（例如：选中“呷哺呷哺” → key: "xiabu", filePrefix: "呷哺呷哺湊湊"）。  
- 读取该客户在 IndexedDB 中已存储的所有周报记录，提取出本地已有的 reportYmd 集合。  
- 将 getRecentWednesdayYmdList(6) 计算出的日期列表过滤掉本地已存在的日期，得到 待补齐的缺失日期清单。  

3. 并发探测与增量落库：
- 若待补齐清单为空，直接弹出：toast("💡 哨兵回报：本地已是最新，无缺失的历史周报")，恢复按钮状态并结束。  
- 针对待补齐清单中的每一个日期（如 20260715），并行发起 Fetch 请求直接拉取绝对路径：
https://bucketforkoagents-1393865189.cos.ap-shanghai.myqcloud.com/sentiment/${customerKey}/${filePrefix}舆情监测${reportYmd}.html
(例如：.../sentiment/xiabu/呷哺呷哺湊湊舆情监测20260715.html)  
- 判定分支 A（HTTP 200 成功）：获取 HTML 纯文本，调用原生的 rewriteEchartsToCdn 和 extractTitleFromHtml 进行清洗格式化，构建周报对象直接追加写入 IndexedDB。  
- 判定分支 B（HTTP 404 或失败）：说明 WorkBuddy 该周未上传报告或文件名不匹配，静默跳过。  

4. 汇总反馈与视图刷新：

- 统计本次成功下载落库的报告数量 successCount。  
- 若 successCount > 0，弹出提示：toast(🎉 哨兵回传：已为您自动补全下载 ${successCount} 份历史舆情周报！)。  
- 最终统一调用 refresh() 瞬间刷新列表看板。  

## 4. 保留手动兜底通道
在控制栏或者折叠区，保留一个隐式的手工 {手动导入本地HTML} 备用链接/按钮，一旦发生网络故障，销售人员依然可以通过原先的 FileReader 流程手动导入文件救急。
