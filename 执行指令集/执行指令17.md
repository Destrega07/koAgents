# 角色与目标
你是一位精通 YAML 语义树分片解析、复杂前端 DOM 状态联动以及现代 CSS Grid 响应式卡片墙构建的顶级全栈开发专家。
我们需要在【7C事实与洞察库】（`moduleKey: "facts7c"`）中，实现三类战略卡片（证据卡、洞察卡、行动假设卡）的批量 Markdown 上传解析、多功能标签筛选以及惊艳的【事实-洞察-行动关系视图】（溯源信任链断点流）。

# 详细重构需求说明

## 1. YAML 语法树分片解析落库 (重构核心上传中继)
- **UI 触发器**：在模块右上角常驻一个 `{导入7C知识卡片}` 按钮。在其旁侧挂载 `<input type="file" accept=".md" multiple hidden>` 控件，支持批量或多选 Markdown 上传。
- **解析器算法（Splitter）**：
  - 用户上传 Markdown 后，通过字符串扫描，以 `## CAR_` 或 `## I` 等卡片主标题作为切片分隔符（泛匹配 `/(?:^|\n)##\s+([A-Za-z0-9_]+)/g`）。
  - **字段精准抽取**：提取每个卡片区块内的 ```yaml ... ``` 代码块，利用原生或高容错的 JavaScript 解析器将其转为结构化 JSON 对象。
  - **卡片类型归流**：识别 YAML 中的 `card_type` 字段进行打标分类：
    - `evidence` 或 `evidence_card` $\rightarrow$ 归为【证据卡】仓库。
    - `insight` 或 `insight_card` $\rightarrow$ 归为【洞察卡】仓库。
    - `action_hypothesis` $\rightarrow$ 归为【行动假设卡】仓库。
  - 数据写入 IndexedDB 独立的 `facts_7c_cards` 仓库中（主键为 `id`），导入成功后弹出 `toast("已成功分片解析并落库 X 张知识卡片")`。

## 2. 三类卡片专属信息展现规格 (卡片墙渲染)
当用户点击顶部三个快速筛选按钮 `{证据卡}`、`{洞察卡}` 和 `{行动假设卡}` 时，页面展示为响应式卡片墙。每种卡片的渲染字段必须高度符合其专业属性：
- **【证据卡】展现字段**：卡片 ID (`id`)、原子结论 (`atomic_conclusion`)、原始出处引用 (`evidence_quote`)、置信度 (`confidence`)、决策主体 (`entity`)。
- **【洞察卡】展现字段**：卡片 ID (`id`)、核心洞察 (`core_insight` 或 `judgment`)、推导传导链 (`transmission_chain`)、置信度 (`confidence`)。
- **【行动假设卡】展现字段**：
  - 卡片 ID (`id`)、行动主题 (`actionTheme`)、目标客户 (`targetCustomer`)。
  - **核心核心机制**：行动机制 (`actionMechanism`)、预期成效 (`targetEffect`)。
  - **外键隐藏锚点（用于联动）**：支撑证据卡 (`supporting_evidence_cards`)、支撑洞察卡 (`supporting_insight_cards`)。
  - **标签**：决策主体 (`decisionOwner`)、成熟阶段 (`maturity_stage`)、观测维度 (`business_dimension`)。

## 3. 【事实-洞察-行动关系视图】多列联动设计 (核心高价值业务)
当用户点击模块上方的 `{事实-洞察-行动关系视图}` 按钮时，页面平滑切换为**从左往右展示“行动假设卡”、“洞察卡”、“证据卡”的三列瀑布流横向排版布局**。
- **行动假设主导联动**：
  - “行动假设卡列”标题下方挂载一个 `<select>` 下拉选框，内容填充当前所有行动假设的 `actionTheme` 文字清单（受观测维度筛选）。
  - 用户从下拉框中选择某个行动假设时，该卡片立刻显示在第一列；同时，系统提取该卡片 YAML 中的 `supporting_insight_cards`（外键数组），在第二列“洞察卡列”中仅展示这些关联卡片；提取其 `supporting_evidence_cards`（外键数组），在第三列中仅展示对应的证据卡。
- **洞察主导联动**：
  - “洞察卡列”标题下方同样挂载一个 `<select>` 下拉选框，内容填充当前所有洞察的标题/主旨文字清单（受观测维度筛选）。
  - 用户选择某个洞察时，“行动假设卡列”直接**清空、隐藏不显示任何卡片**；“洞察卡列”显示该洞察卡；“证据卡列”通过该洞察卡内的 `supporting_evidence_cards` 外键关联，仅展示与其挂钩的证据卡。

## 4. 全局“观测维度”下拉筛选器集成
- 在模块页面最顶部常驻一个叫 **“观测维度”** 的全局下拉筛选器（`<select class="select">`），固定选项为：`["All", "Channel", "Category", "Consumer", "Customer", "Competitor", "SupplyChain", "Pricing", "Brand"]`。
- **联动效应**：
  - 在卡片墙模式下，切换维度将直接过滤卡片中的 `business_dimension` 字段，实现动态缩减。
  - 在关系视图下，“行动假设列”和“洞察列”标题下方的**两个联动下拉列表中的选项清单，也必须同步受到该“观测维度”的严格过滤与实时裁剪**。

# 交付标准
1. 三类卡片的 YAML 数据解析 100% 准确，不发生跨行或字段错漏。
2. 关系视图下的三列级联联动完美丝滑，选择行动假设或洞察时，右侧卡片墙能够无缝实现拓扑追溯展现。
