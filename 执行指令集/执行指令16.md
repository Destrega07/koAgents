# 角色与目标
你是一位精通原生 JavaScript DOM 渲染、FileReader 文件异步流读取以及 iframe 沙盒隔离技术的资深全栈工程师。
我们需要全新跑通【合作客户管理】主页面下的【 KA 关键动作预测与机会洞察】子模块（对应 `moduleKey: "ka_actions"`）。该模块要求支持手工导入由 Openclaw 生成的深度洞察 HTML 报告，在本地进行“最新快照覆盖落库”，并实现点击或选中客户时直接在页面正中央无损渲染报告。

# 详细开发规格与重构指令

## 1. 离线持久化数据结构设计 (基于 storage.js 规范)
本模块采取“最新快照覆盖策略”，每个客户在模块中永远只保留最新的一份报告。
- 请在 `storage.js` 或数据管理层中，当调用 `setModuleData(customerId, "ka_actions", data)` 时，确保其数据结构为扁平的对象字典，而不是数组：
  ```json
  {
    "customerId": "当前选中的客户ID，例如：cust_haidilao",
    "moduleKey": "ka_actions",
    "updatedAt": "ISO时间戳",
    "reportTitle": "自动解析捕获的报告标题",
    "htmlContent": ""
  }
  ```

- 每次针对同一客户导入新报告时，直接使用 put 机制整体覆写该复合键下的记录，实现物理层面的“覆盖更新”。

## 2. 模块控制栏与手工导入联动 (重构 renderKaActions)
重构 app.js 中的 renderKaActions 视图函数：

- 布局对齐：模块顶部复用系统的标准长条卡片面板，包含“客户选择”组件。

- 手工导入触发器：在控制栏右侧常驻一个高级感视觉的 {导入关键动作与洞察报告} 按钮（对齐系统经典的红白/灰色调视觉风格）。

- 文件读取流：在其旁侧隐式挂载一个文件控件 <input type="file" accept=".html,.htm" hidden>。

- 智能落库：当销售人员选中 HTML 文件后，利用 FileReader 异步将其读取为纯文本字符串。利用正则解析出其中的 <title> 文本（如“红火先知 | 海底捞集团洞察报告”）存入 reportTitle，随后将全量文本作为 htmlContent 执行覆盖式落库存储，并弹出 toast("最新大客户机会洞察报告已覆盖更新并成功落库") 随即刷新页面。

## 3. 页面直接无损渲染沙盒看板 (实现点击 Logo / 选中客户即显)
当页面选中了特定的客户（state.selectedCustomerId 存在）时，模块主渲染区域的交互规格如下：

- 无数据占位态：如果当前客户在 IndexedDB 中查不到任何 ka_actions 记录，页面主区域渲染精美的 Markdown 占位提示：“暂无动作预测与机会洞察报告。请上传该大客户的 HTML 洞察报告以激活看板。”。

- 页面内直接嵌入（100% 无损还原）：如果该大客户存在已导入的报告，严禁使用弹窗拦截。请直接在页面主卡片面板内部，挂载一个自适应高度的 <iframe> 组件：

```HTML
<iframe sandbox="allow-scripts" style="width: 100%; height: calc(100vh - 240px); border: none; border-radius: 12px; box-shadow: 0 4px 6px -1px rgba(0,0,0,0.02);"></iframe>
```

- 注入与页边距对齐：

  - 将从数据库中提取出的 htmlContent 纯文本字符串，直接赋值给该 iframe 的 srcdoc 属性（iframe.srcdoc = data.htmlContent），使其内部所有的微渐变、卡片布局和核心表格在页面中原生无缝舒展。

  - 容器左右页边距严格保持系统规范的 24px，确保页面整体的呼吸感与大牌专业度。

## 交付标准
1. 切换不同客户时，页面内的 iframe 看板能瞬间完成平滑刷新，并 100% 还原报告里的所有精美图表和文字。
2. 再次导入新报告时，旧报告被完美物理覆盖，刷新后依然保持单份最新快照的精简展现。
