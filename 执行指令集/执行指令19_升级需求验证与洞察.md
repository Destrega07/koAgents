# 详细重构需求说明

## 1. 顶部控制栏双选升级（支持合作者多选）
请修改 `renderNeedInsight` 的顶部控制栏组件：
- **合作者多选组件**：在“客户选择”下拉选框右侧，新置一个 **“合作者选择”** 下拉选框。点击后弹出一个带有复选框（Checkbox）的轻量级下拉列表，允许销售人员在上传访谈文档前，勾选本客户名下的多位合作者名称。
- **动态基础信息代入**：点击 `{深度解析}` 并调用 `<客户需求深度洞察小军师>` 返回后，生成的报告卡片中 `### 基础信息` 模块下的 `- **访谈对象**：` 字段，必须直接强行显式渲染出用户刚才在顶部勾选的所有合作者全名合集（以逗号分隔，如：Icy总, 发言人二）。

## 2. 离线多报告卡片墙机制重构 (持久化层升级)
- **多报告落库**：重构 IndexedDB 存储。同一个客户名下的 `moduleKey: "need_insight"` 不再是单一对象，升级为多卡片数组形式。每次点击深度解析生成新报告，均自动在其数组中追加一个包含唯一 `subReportId`、`createDate`（自动提取当前系统 YYYY-MM-DD）的快照对象。
- **卡片墙三键组件化渲染**：
  - 页面主区域展现为报告卡片墙，每张卡片独立包裹。卡片右上角带三个精致的小图标按钮：`{折叠}`、`{展开}`、`{删除}`。
  - **折叠状态（Collapsed）**：页面卡片收缩，仅高雅呈现 `### 基础信息`（客户企业、访谈对象、访谈质量）以及卡片生成日期。
  - **展开状态（Expanded）**：舒展显示包含口述、支持、成果、个人需求及饮料角色在内的完整 Slot 矩阵。
  - `{删除}` 按钮点击后，触发 `confirmModal` 确认后从本地数据库数组中将该 `subReportId` 物理剔除。

## 3. 隐藏复选框与【需求合并解读】状态机点亮控制
- **隐藏悬停选框**：在每张多报告卡片的左上角，隐式挂载一个前端复选框（`<input type="checkbox" class="card-selector-cb">`）。该小选框默认通过 CSS 的 `opacity: 0; pointer-events: none;` 隐藏。**只有当用户鼠标悬停（Hover）在特定卡片上时**，该选框才触发 `opacity: 1; pointer-events: auto;` 浮现出来。
- **按钮点亮状态机**：
  - 在顶部 `{深度解析}` 按钮的右侧紧邻并排新增一个红色主按钮：**`{需求合并解读}`**。
  - **静默置灰**：该按钮初始化及平时默认处于 `disabled = true` 的静默灰色状态。
  - **激活条件**：当前端监听到用户通过卡片左上角复选框勾选的卡片数量 **$\ge 2$ 张** 时，瞬间点亮该按钮（`disabled = false`），转呈现为醒目的可口可乐红。

## 4. 召唤新智能体与联邦合并报告展示区域
- 当用户选中多张卡片并点击 `{需求合并解读}` 按钮时，前端执行以下数据联邦提取：
  - 读取出被勾选的这 2 张或多张卡片内**已经存在、解析清洗干净的各层需求 Slots 数组和文本块**。
  - 将这些多卡数据组合为一个名为 `merge_federal_context` 的 JSON 对象。
  - 发起Fetch流式通信，**主特许召唤全新智能体 —— `<需求合并交叉洞察专家>`（Bot ID: 7658141846765928474）**。
- **不覆盖陈列原则**：AI 返回的合并洞察报告，必须通过打字机组件流式动态渲染在**当前卡片墙最下方的全新专属合并区域中**，绝对允许覆盖、干扰或污染上方的历史卡片墙基本盘。
