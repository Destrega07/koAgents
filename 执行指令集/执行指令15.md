# 角色与目标
你是一位精通原生 JavaScript DOM 高级渲染、文件流读取（FileReader）以及 iframe 状态控制的资深全栈工程师。
我们需要彻底跑通【合作客户管理】主页面下的【 KA 每周舆情监测】子模块（对应 `moduleKey: "ka_sentiment"`）。该模块要求支持手工导入完整的舆情 HTML 周报文件，在本地持久化并完美渲染。

# 详细重构需求说明

## 1. 控制栏扩展与 HTML 报告手工上传 (DOM 构建)
重构 `app.js` 中的 `renderKaSentiment` 视图挂载函数：
- **顶部控制栏布局**：复用系统的标准长条卡片面板（`renderCustomerContextBar`），包含“客户选择”下拉选框。
- **手工导入组件**：在控制栏右侧常驻一个 `{导入每周舆情报告}` 按钮（样式对齐系统经典红白风格）。在其旁侧隐式挂载一个支持限制扩展名的文件选择控件 `<input type="file" accept=".html,.htm" hidden>`。
- **文件读取与落库流**：
  - 用户点击 `{导入每周舆情报告}` 触发文件选择。当选中一个 Openclaw 輿情 HTML 周报时，通过 `FileReader` 对象的 `readAsText(file)` 异步将其通篇源代码读取为纯文本字符串[cite: 40, 41]。
  - **标题自动解析**：通过正则表达式（如 `/ScrimRe = /<title>([\s\S]*?)<\/title>/i`）自动抓取 HTML 中的标题内容作为周报卡片的主标题。
  - 读取成功后，追加写入 IndexedDB 的 `module_data` 中该客户名下的 `ka_sentiment` 数组里，并触发 `toast("舆情周报手工导入并离线落库成功")`。

## 2. 精美舆情卡片看板网格流设计 (AS-IS 缩略态看板)
- **卡片网格渲染**：模块主体展现为一个标准的响应式网格看板。
- **历史周报卡片**：从数据库中读出该客户名下的所有舆情周报，每个周报渲染为一个独立的精致卡片组件：
  - 卡片头部带有 Openclaw 经典的深色调图标，中间清晰展示自动捕获的周报标题（如：汉堡王中国网络舆情周报...）[cite: 40]。
  - 卡片下方以 muted 小字号灰字展示“手工导入时间：YYYY-MM-DD”。
  - 销售人员在电脑端鼠标悬停在卡片上时，卡片伴随有平滑的微上浮与投影加深特效（`transform: translateY(-2px); transition: transform 0.2s ease;`）。

## 3. iframe 隔离无损全屏弹窗/抽屉展现 (TO-BE 全显态)
- 当销售人员点击上述任意一张舆情周报卡片时，系统触发全屏遮罩大弹窗（或可复用系统已有的 `openModal` 组件进行扩容宽高限制）：
  - 弹窗正中央挂载一个高宽均自适应 100% 撑满的沙盒隔离组件：
```html
    <iframe sandbox="allow-scripts" style="width:100%; height:calc(100vh - 160px); border:none;"></iframe>
    ```
  - **安全注入**：将读取出来的 HTML 源码字符串，直接赋值给该 iframe 的 `srcdoc` 属性（`iframe.srcdoc = report.htmlContent`），实现 100% 格式无损还原，且其内部的 ECharts 脚本能够完好运转。
  - 在大弹窗右上角提供一个精致的 `{关闭报告}` 按钮，点击后销毁 iframe，销毁内存，优雅回退到看板主页。

# 交付标准
1. 选中客户后点击导入，能够成功读取并持久化大体积 HTML 文件。
2. 看板卡片错落有致，点击卡片能瞬间拉起无损的 iframe 周报视图，内部所有图表排版节奏完美。
