# 详细重构需求说明

## 1. 跨模块多源数据联邦组装 (打通 renderNeedInsight 至 renderValueProp 管道)
重构【需求验证与洞察】最下方的 `{围绕需求创造价值主张}` 点击事件，当销售点击该按钮时，前端需执行以下“联邦数据打包”：
- **数据源 A（勾选的需求）**：遍历并收集当前页面中所有被用户勾选（`checked === true`）的独立 Slot 需求信息，并必须保留其所属的类型（如：“支持需求”、“成果需求”或“个人需求”）。
- **数据源 B（不可编辑文本块）**：获取当前页面的 “饮料角色” 与 “战略机会提示”（如有）。
- **数据源 C（客户档案）**：从 IndexedDB 中读取当前 `state.selectedCustomerId` 对应的**全量客户档案字段信息**。
- **打包分发**：将上述 A、B、C 三方数据深度整合为一个名为 `pipeline_context` 的结构化 JSON 对象。将其作为 `message` 参数包，调用流式或非流式接口发送给 `<价值主张创造小参谋>`（Bot ID: `7552194182392512566`）。
- **页面跳转**：发起请求的同时，系统立即无缝更改 Hash 路由指针，自动平滑跳转至【价值主张创造】模块页面。

## 2. 核心算法：高鲁棒性 Markdown 模块切片器 (Parser 重构)
在【价值主张创造】模块页面中，系统需要接收并拆解小参谋返回的标准 Markdown 报告，精准放入对应的四大前端可视化内容模块中。请使用带防御性隔离边界的正则或解析逻辑剥离以下四大区块：
1. **方案及价值主张罗列**：提取 `### 一、方案及价值主张罗列` 下的内容。
   - **Slot 化分拆**：该区块内通常包含多个方案（如：方案一、方案二）。你必须使用正则或列表解析器，**将每个方案及对应的价值主张拆分到一个独立的 Slot 小卡片组件中**。
   - **交互组件**：每个 Slot 卡片左侧常驻一个复选勾选框（Checkbox），默认状态为未勾选。
2. **方案独特性分析**：提取 `### 二、独特性分析` 下的内容，渲染为美观、不可勾选的文本面板。
3. **总结表格**：提取 `### 三、价值效率与投资回报评估`（或包含表格特征的区块），直接通过原生 HTML 的 `<table>` 体系渲染为专业美观的数字化矩阵。
4. **价值主张合成**：提取报告最后的总结段落，渲染为独立的陈列面板。
*注：必须支持用户直接在上述各个模块的内容框上进行手工编辑与改动。*

## 3. 页边距呼吸感与高亮联动样式对齐
- **统一页边距**：全面继承【需求线索】和【需求验证与洞察】的顶级视觉调性。模块容器左右 Padding 严格锁定为 `24px`，确保卓越的页面呼吸感。
- **高亮联动 (Highlight)**：当用户在“方案及价值主张罗列”中勾选了某个方案 Slot 卡片时，通过 `onchange` 事件，为该卡片容器平滑加上 `.is-highlighted` 激活样式（微调背景色为淡红调 `rgba(244, 0, 9, 0.04)`，边框变为 `1px solid rgba(244, 0, 9, 0.3)`）。

## 4. 底部并排提示条与点亮状态机控制
在【价值主张创造】页面报告的最下方，常驻一个带有灰色背景与虚线边框（圆角 `10px`，边框 `1px dashed #e5e7eb`）的长条提示框（Callout）。
- **提示语简化与固化**：在提示框左侧展现以下固化文案：
  > “💡 **提示**：请校准报告内容，勾选你认同的方案，点击右侧的【万能剧本提案】按钮创造提案。”
- **并排按钮**：将 `{万能剧本提案}` 按钮移至该提示语的右侧，临近对齐，共同处于 Flex 弹性布局流中。
- **点亮状态机控制**：
  - **非激活状态**：当用户未勾选任何方案 Slot 卡片时，该按钮必须处于非激活状态（`disabled = true`），呈现灰色。
  - **激活状态**：只要用户勾选了任意一个（或多个）方案小卡片，该按钮立刻解除锁定（`disabled = false`），转呈现为高亮的可口可乐经典红。勾选状态变化时需实时重算。
