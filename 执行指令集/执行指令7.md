# 详细优化指令

## 1. 扣子返回内容去重与清洗算法升级 (后端与提取策略层)
针对目前 `app.js` 中 `sliceNeedInsightMarkdown` 函数解析 Coze 回传数据时可能出现的“重复解析两份报告”现象，请全面重构 Markdown 提取与清洗机制：
- **引入单次截断逻辑**：参考 Coze API 在 message 列表中可能由于机制（如 `tool_response` 与 `answer` 并存）导致内容成倍重复输出的规律。解析器在深度扫描传入的 Markdown 大文本时，一旦匹配到第二个相同的主标题（如再次匹配到 `# 需求深度洞察分析报告`），必须**立即执行安全截断（Slice 过滤）**，强行舍弃该标记点之后的全部重复追加文本。
- **优先策略对齐**：确保数据清洗严格遵循“P0 (answer) -> P1 (tool_response)”的黄金选择法则。在获取 `report` 原始文本流时，优先且唯一提取单次完整闭环的内容。

## 2. 需求 Slot 小卡片视觉排版优化与激活高亮 (UI/UX 升级)
请升级 `renderNeedInsightSlotSection` 函数动态生成的 Slot 卡片（支持、成果、个人需求）的样式：
- **排版对齐风格**：全面对齐【需求线索】模块的高级感。每个 `#### 需求` 二级标题左侧，加上一条宽 `4px`、高 100% 对齐的**可口可乐经典红（`#f40009`）实体视觉轴线边框**。
- **留好页边距**：整个报告卡片容器左右内边距（Padding）设定为 `24px`，确保页面呼吸感，消除文字贴边的压迫感。
- **状态高亮联动 (Highlight)**：
  - 修改勾选框（Checkbox）的 `onchange` 事件监听。
  - 当需求小卡片被用户勾选（`slot.checked === true`）时，为该 `.slot-card` 容器动态追加一个高亮类（如 `.is-highlighted`）。
  - **CSS 表现**：高亮时卡片背景色平滑渐变为极淡的可口可乐红调（如 `rgba(244, 0, 9, 0.04)`），并且边框变更为 `1px solid rgba(244, 0, 9, 0.3)`，为用户提供极其明确的视觉反馈；取消勾选时平滑复原。

## 3. 提示语精简与右下角操作按钮布局重构
- **提示语简化**：将图中原有的多行复杂声明，彻底替换为以下精简一行的文本定义：
  > “💡 **提示**：请校准报告内容，勾选你打算支持的客户需求，点击右侧红色按钮，系统将围绕需求设计价值主张。”
- **布局流对齐**：
  - 彻底废弃原本置于容器最下方、占据一整行的 `{围绕需求创造价值主张}` 独立按钮行。
  - 将此红色主按钮直接移动到上述**精简提示语的右侧**，使用 Flex 布局（`display: flex; justify-content: space-between; align-items: center; gap: 16px;`）让提示词与按钮并排在同一个长条提示框（Callout）中。
- **点击状态机控制（点亮逻辑）**：
  - **默认置灰**：初始渲染或未满足条件时，该红色主按钮必须处于 `disabled = true` 状态，视觉透明度降低。
  - **点亮变活**：只有当“需求验证与洞察解读报告”已成功解析渲染存在，**且**用户在页面中**至少勾选了任意一个需求 Slot 小卡片**时，该按钮才解除锁定（`disabled = false`），瞬间点亮恢复可点击状态。用户点击或取消勾选卡片时，系统需实时重算并刷新该按钮的启用状态。
