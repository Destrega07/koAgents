# 角色与目标
你是一位顶级的全栈工程师。现在我们需要大幅更新现有的【可口可乐销售赋能中心】项目。
我们将把原有的“多智能体单页聊天室”重构为“销售数字化看板 + 智能体异步流水线”架构。
本阶段的任务是：重构前端 UI 骨架、引入 IndexedDB 本地存储底座、并实现系统级的备份恢复功能。

# 核心架构变更
1. 后端 (server.js)：保持原有的 Coze API 转发代理功能不变，但移除原有的单页 index.html 路由，改为支持全新的、模块化的前端页面。
2. 前端持久化：完全放弃无状态模式，引入浏览器 IndexedDB 作为唯一的本地数据存储底座，用于存储所有客户档案、合作者档案、各模块的结构化数据。
3. 系统级备份：在页面显著位置提供 {导出备份}（将 IndexedDB 完整数据序列化为 JSON 文件下载）和 {导入恢复}（读取 JSON 文件并写入 IndexedDB）功能。

# 详细重构需求说明

## 1. UI 骨架重构 (index.html & 样式升级)
- 废弃原有的“左侧聊天列表+右侧聊天框”单页布局。
- 升级为“双层导航”的现代管理后台风格（保持可口可乐 fresh, clean, professional 的红白/灰色调视觉风格）：
  - 侧边栏/顶部主导航：分为【潜客开发】和【合作客户管理】两大主页面，以及系统级按钮（{导出备份}、{导入恢复}）。
  - 子模块导航（Tab 切换或滚动锚点）：
    - 【潜客开发】包含子模块：【客户档案】、【合作者档案】、【需求线索】、【需求验证与洞察】、【价值主张创造】、【提案剧本】。
    - 【合作客户管理】包含子模块：【客户档案】、【合作者档案】、【KA每周舆情监测】、【KA关键动作预测与机会洞察】、【7C事实与洞察库】。

## 2. IndexedDB 本地数据底座设计 (storage.js)
请新建 `storage.js`，设计并初始化 IndexedDB 数据库，包含以下核心数据仓库（Store）：
- `customers` (客户档案)：存储潜客/合作客户的卡片字段。
  - 关键字段包括：id, name, status(待提案/已提案/已成交/被拒绝等), infoCompleteness(信息完备度), type(潜客/合作客户), 展开状态所需的所有餐饮/饮料业务字段(店数、营收等), 掌门人发言要点, 发展策略等。
- `collaborators` (合作者档案)：存储合作人员信息。
  - 关键字段包括：id, customerId(关联客户), name, position, relationStatus(首次接触/存有戒心/初步信任等), 置信度评分, 沟通建议等。
- `module_data` (模块流水线数据)：用于存储各阶段智能体返回并解析后的结构化数据（如需求 Slot、价值主张 Slot 及其勾选状态）。

## 3. 系统级功能实现
- 实现 {导出备份} 按钮：一键读取 IndexedDB 的所有 Store 数据，打包成 `.json` 文件并触发浏览器下载。
- 实现 {导入恢复} 按钮：支持用户上传 `.json` 备份文件，清空当前 IndexedDB 并将备份数据完整恢复，恢复后自动刷新页面渲染。
