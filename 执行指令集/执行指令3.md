# 详细重构指令

## 1. 顶部【客户选择长条卡片】布局像素级重构
请修改 `renderNeedLeads` 函数中关于客户选择面板（`renderCustomerContextBar` 或相关 DOM 节点）的 HTML 结构与样式：
- **容器宽度**：保持变动后的面板总宽度与下方“分析报告区域”100% 严格对齐、左右同宽。
- **文字与组件重组**：
  - 将原文字 “上下文客户” 变更为 “客户选择”。
  - 将客户选择的 `<select>` 下拉选框直接移动到 “客户选择” 文字的右侧，紧邻文字。
  - 将 `{发送客户档案}` 按钮移动到该 `<select>` 下拉选框的右侧，紧邻下拉选框。
  - 以上三者（文字、下拉框、发送按钮）使用 `display: flex; align-items: center; gap: 12px;` 形成左对齐的流式布局。

## 2. 引入联调弹窗的“五连击隐藏彩蛋”机制
为了不影响生产环境的丝滑体验，同时保留极客调试能力，重构 `{发送客户档案}` 的点击事件逻辑：
- **状态变量**：在 `state` 或模块作用域内，引入一个连击计数器 `clickCount = 0` 和最后点击时间戳 `lastClickTime = 0`。
- **隐藏判定逻辑**：用户必须在 **2 秒内连续点击模块页面左上角的 “潜客开发” 文字 5 次**，系统才激活本地调试模式（可在控制台或界面给个极其隐蔽的淡色小点提示）。
- **执行分支**：
  - **默认直接发送**：如果直接点击 `{发送客户档案}` 按钮，系统默认跳过任何拦截，直接打包 JSON 并请求 Coze 端 API。
  - **联调弹窗触发**：只有当上述隐藏彩蛋被触发激活后，点击 `{发送客户档案}` 才会弹出第一阶段开发的 “JSON 预览与一键复制弹窗”。

## 3. 分析报告渲染区域视觉表现力重构 (对齐并超越高级感样本)
当解析并渲染 Coze 返回的 Markdown 报告文本时，请重构前端 CSS 样式或动态生成的 DOM 节点，打造极高视觉吸引力的版式：
- **卡片化分层布局**：
  - 报告根容器使用纯白背景（`#ffffff`），四周加上非常柔和的多层微阴影（`box-shadow: 0 4px 6px -1px rgba(0,0,0,0.05), 0 10px 15px -3px rgba(0,0,0,0.03);`），圆角调整为 `16px`。
  - 报告内部的每一个二级标题（如：`## 一、 客户理想`，`## 二、 客户现实`）渲染为一个独立的“大视觉区块”，区块之间保持 `24px` 的大内边距（Padding）和外边距（Margin），形成呼吸感。
- **可口可乐红（Coke Red）视觉锚定**：
  - 每一个大 H2 二级标题左侧，必须加上一条宽 `4px`、高 100% 对齐的**可口可乐经典红（`#f40009`）实体视觉锚定边框（Left Accent Border）**，并配合 `padding-left: 12px;`。
  - 二级标题的文本颜色升级为带有高级感的深炭黑（`#1f2937`），字重（Font-weight）为 `800`。
- **信息排版节奏与关键字高亮**：
  - 报告中的 `### 积极现实` 展现为浅绿调性的精致标签或副标题，`### 挑战现实` 展现为浅红/浅橘调性。
  - 无序列表的各项条目（`-` 或 `*` 转换后的 `<li>`），行高（Line-height）统一放大到 `1.7`，文字颜色采用高易读性的中深灰（`#374151`）。
  - 使用正则表达式自动识别或确保 Markdown 解析器将条目中的 **加粗文本（`**文本**`）** 统一替换为带有可口可乐红语义的高亮加粗，增强销售抓取关键词的效率。
