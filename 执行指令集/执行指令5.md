# 角色与目标
你是一位精通现代 CSS 弹性/网格布局及企业级 CRM 数据管道开发的全栈工程师。
我们需要对【合作者档案】模块进行“视觉重构（高级感板式）”与“核心底层数据流（多轮对话延续性）”的重大升级。

# 核心重构需求说明

## 1. 桌面端（PC）布局升级：多列响应式网格 (Grid Layout)
- **卡片排列**：修改现有的 `.grid` 或包含合作者卡片的容器样式。在网页电脑端（屏幕宽度 $\ge$ 1024px）展示时，由单列长条形布局重构为 `grid-template-columns: repeat(auto-fill, minmax(450px, 1fr));`。
- **效果说明**：使横向能够自动并行容纳 2 到 3 张合作者档案卡片，充分利用屏幕横向空间，消除空白。

## 2. 合作者卡片内部组件级 UI 像素重构
请对合作者卡片（编辑、展开、折叠状态）进行以下视觉层级优化：
- **置信度字段同行并排（Inline Typography）**：
  - 移除原先单独占一行的“置信度（性格特征）”、“置信度（沟通风格）”、“置信度（决策偏好）”等标签。
  - 将置信度分值直接移动到其对应字段内容的**右侧，保持在同一行**。例如：
    > **性格特征**：I（影响型）+ C（严谨型）混合型 `(置信度: 85%)`
  - 置信度文字使用括号包裹，字体大小调小（`12px`），颜色使用小吃看板主色调中的中灰或 muted 灰色。
- **关系状态按钮扁平精致化**：
  - 缩减 `{首次接触}`、`{存有戒心}` 等 6 个状态按钮的尺寸，将其 `padding` 缩减为精致的 `4px 8px`，字号调小至 `12px`。
  - 移除扎眼的边框，改为轻量级的胶囊标签（Capsule Badges）。选中时高亮，未选中时保持柔和的背景色，使其不再显得过大和突兀。
- **Top 3 沟通建议版式视觉重构 (核心高价值区域)**：
  - 将三条沟通建议从纯文本列表升级为**“独立视觉区块（Callout Cards）”**。
  - 每一条建议包裹在一个带有极淡底色（如可口可乐红的 3% 透明度浅红 `#fff5f5` 或浅灰色）的卡片里，左侧加上 `2px` 实色边框。
  - 标签化加粗展示 `【适用场景】`、`【策略】`、`【话术示例】`，行高设定为 `1.6`，提升销售的阅读和抓取效率。

## 3. 多轮对话“延续性”联邦数据流升级 (全栈架构层)
- **支持一客多伴**：解除原先“一个客户只能绑定一个合作者档案”的代码限制。在 `storage.js` 和存储逻辑中，确保 `collaborators` 仓库通过外键 `customerId` 能够检索、列出并关联**多条**不同的合作者记录。
- **历史上下文打包发送**：
  - 当销售在某个已存在 AI 解析数据的合作者卡片上，再次上传新一轮对话录音并点击 `{解析}` 时，系统**不得只发送新文本**。
  - 前端需将当前 IndexedDB 中**该合作者现有的历史评估字段**（当前的性格特征、沟通风格、决策偏好、对话分析累积样本数等）组装为一个名为 `history_context` 的 JSON 对象。
  - 将该 `history_context` 对象作为系统前置 Prompt 提示词，与用户新上传的对话录音转文本素材**合并拼装成一条完整的新 message** 投喂给 Coze 代理接口，让扣子端智能体感知过往评价，进行增量更新。
