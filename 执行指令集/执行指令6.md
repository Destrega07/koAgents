# 详细重构需求说明

## 1. 顶部控制栏与文件分发联动 (HTML/DOM 重构)
- **UI 布局**：在 `renderNeedInsight` 顶部引入标准长条卡片面板，通过 `state.selectedCustomerId` 动态渲染“客户选择”下拉框，右侧并排紧邻 `{上传文档}` 按钮和主高亮按钮 `{深度解析}`（保持系统经典红白视觉风格）。
- **自动化参数打包**：
  - 点击 `{深度解析}` 时，系统必须自动捕获当前选定的客户名称（如 `customer.name`）。
  - 将客户名称与用户通过 `{上传文档}` 缓存的录音文字文档字符串合并打包，通过 POST 请求发送给后端的 `/api/chat` 流式代理接口。

## 2. 核心算法：Markdown 深度切片与 Slot 状态机设计
接收到小军师返回的报告后，前端不能整体粗暴呈现，必须建立高鲁棒性的 Markdown 分解器（Parser），将其内容精准剥离、放入对应的四个模块中：
- **基础信息、饮料角色、需验证的关键假设、战略机会提示**：直接渲染为美观、不可勾选的文本区块。
- **三大核心需求 Slot 化拆解（支持需求、成果需求、个人需求）**：
  - 扫描 Markdown 下的对应四级标题（如 `#### 支持需求`）。
  - 如果该模块里存在多条需求信息（以数字列表 `1. 2. 3.` 或无序列表 `-` 分隔），**必须将每条需求独立拆分到一个独立的 Slot 组件中**。
  - **组件形态**：每个 Slot 为一个独立的轻量级卡片，内含：
    1. 一个前端勾选框（`<input type="checkbox">`），默认状态为 `unchecked`（未勾选）。
    2. 需求具体内容文本（允许用户直接在卡片上双击或点击进行手工录入与二次编辑修改）。

## 3. 页面内提示语固化与 Human-in-the-Loop 交互
- 在页面报告区域和最下方操作纽之间的合适醒目位置，固化以下设计精美、带有弱视觉背景的系统提示文本：
  > “💡 **提示**：请仔细校准内容，并勾选你打算在后续方案中予以满足的目标需求，点击页面最下方的【围绕需求创造价值主张】按钮，系统将为你联动创造精准的价值主张方案。”
- 在容器最下方常驻高亮主按钮：`{围绕需求创造价值主张}`。

## 4. 跨智能体数据流转与持久化落地 (落库 IndexedDB)
- **多选数据采集**：当销售点击 `{围绕需求创造价值主张}` 按钮时，前端必须遍历并收集当前页面中所有**被用户勾选（`checked === true`）的独立 Slot 需求信息**。
- **数据结构打包**：将收集到的已勾选需求数组，与该页面的“基础信息”、“饮料角色”、“战略机会提示”合并压缩为一个完整的 JSON 数据包。
- **状态落库**：调用 `setModuleData(cid, "need_insight", payload)` 将该多选状态数据深度持久化存储到 IndexedDB 数据库中。
- **自动锚点跳转**：保存成功后弹出 `toast("选定需求已锚定，正在生成价值主张流水线...")`，并在 400ms 内自动更新浏览器 Hash 指针，无缝横向平滑跳转至下一个子 Tab ——【价值主张创造】模块页面。
