# KA管理与潜客开发智能系统介绍文档

## 1. 系统定位

KA管理与潜客开发智能系统是一套面向大客户（KA）与潜客（Lead）场景的“研究—洞察—提案—复盘—持续监测”一体化工作台。系统将传统“零散资料 + 临时脑暴 + 单次提案”的工作方式，升级为：

- 可持续沉淀的结构化知识（证据卡 / 洞察卡 / 行动假设卡）
- 可复用的洞察-提案流水线（需求验证 → 价值主张 → 提案剧本）
- 可追溯、可编辑、可导出的交付物（报告、话术剧本、长图）
- 可长期运营的监测与预警机制（舆情周报、KA关键动作预测）

该系统面向KA管理团队，用于统一方法论、提升提案一致性、加速新人复制与规模化运营。

## 2. 设计理念

### 2.1 从“输出”到“证据链”

系统将客户研究、行业研究与舆情信息转化为结构化卡片，并通过关系视图把“证据 → 洞察 → 行动假设”贯通，使提案从“有观点”升级为“有证据、有逻辑、有边界条件”的可交付成果。

### 2.2 从“单次提案”到“持续经营”

系统不仅生成提案，还围绕 KA 客户建立长期经营闭环：持续监测（周报）、识别策略、关键动作与机会、沉淀可复用洞察，形成“持续迭代”的客户经营资产。

### 2.3 从“工具堆叠”到“可运营工作台”

系统提供统一的 UI、统一的数据底座、统一的导入与持久化策略，减少“文件散落 + 信息难复用”的运营成本。

### 2.4 本地与上云双路径应用方案

- 可选用本地数据存储策略（浏览器端 IndexedDB），实现安全与速度并举应用效果。并提供备份/恢复机制，保障资料与产物可控、可迁移、可长期积累。
- 可选用企业私有云部署方案，持续将KA管理数据沉底为企业资产。

## 3. 整体框架（模块全景）

系统以两条主线组织能力：

1) 潜客开发·普客管理（Lead）：从需求到提案的高效率流水线  
2) 战略客户管理（KA）：从监测到行动机会的持续经营闭环

同时由 7C 事实与洞察库承载“结构化知识底座”，为两条主线提供统一的证据与洞察来源。

## 4. 在客户管理上的核心价值

### 4.1 提升“提案精准度”

- 累积真实语料，持续完善客户档案、合作者画像，深度洞察多层次需求
- 合作者画像包含性格特征、沟通风格、决策偏好的心理学洞察
- 提案精准指向合作者
- 大幅降低因个人经验差异导致的质量波动

### 4.2 提升“提案效率”

- 资料导入后可复用，不必每次从零开始整理
- 价值主张与提案剧本一键生成，节省多轮拼贴与重写时间
- 导出html/PNG长图版提案，直接支撑外部汇报与内部对齐

### 4.3 提升“可追溯与可复盘”

- 证据—洞察—行动链路可回溯
- 卡片字段支持边界条件与风险继承，避免过度推断
- 便于复盘：哪些证据支撑了哪些洞察，哪些洞察驱动了哪些动作

### 4.4 提升“持续经营与预警能力”

- KA《舆情监测周报》、《动作预测与机会洞察月报》自动推送
- 关键动作预测与机会洞察带有置信度评估，自动根据新证据进行贝叶斯更新

## 5. 模块介绍（按业务场景）

### 5.1 潜客开发·普客管理（Lead）

#### (1) 需求挖掘陪练

- 面向一线销售的“对话陪练”入口
- 通过结构化分段输出，辅助完成提问、澄清、验证与总结
- 适用于新人训练、话术演练、会前准备

#### (2) 需求验证与洞察

- 将客户描述的需求转换为更可执行的洞察表达
- 支持展示关键洞察与机会线索（从结构化字段与 Markdown 兼容提取）

#### (3) 价值主张创造

- 围绕需求生成多方案“价值主张 Slot”
- 支持勾选方案、编辑落库、以结构化方式进入下一步提案剧本

#### (4) 提案剧本

- 将多源信息联邦打包，生成可直接对外讲述的“万能剧本提案”
- 支持流式生成、可编辑、持久化、导出 TXT/PNG 长图

### 5.2 战略客户管理（KA）

#### (1) 客户档案

- 统一沉淀客户的基础信息、覆盖区域、关键资料等
- 支持 CSV/Markdown 批量导入，便于快速建档与迁移

#### (2) KA 每周舆情监测

- 监测客户动态
- 监测活动效果
- 监测机会转强或转弱的信号

#### (3) KA 关键动作预测与机会洞察

- **先人一步**发现**可靠**的战略级机会
- 为了做到**先人一步**，使用多证据多阶推理
- 为了做到**可靠**，机会评估可追溯，可反证，并持续更新

#### (4) KA战略咨询与业务合作双提案架构

- 战略咨询提案，让KAM成为客户公司战略共创者，上牌桌定规则
- 业务合作提案，实现机会到业务成果的转化


### 5.3 7C 事实与洞察库（知识底座）

#### (1) 卡片类型

- 证据卡（Evidence）：事实、数据、原文引用与边界
- 洞察卡（Insight）：跨证据推导、置信度、机会线索
- 行动假设卡（Action Hypothesis）：面向可执行动作的假设与验证路径

#### (2) 导入与兼容

- 批量导入 Markdown 文件，自动切片识别卡片并解析 YAML
- 兼容“YAML + Markdown 混合字段”的卡片写法，导入时回填缺失字段
- 主键统一使用全局唯一 id，避免跨文件短号覆盖与关系断裂

#### (3) 关系视图

- 三列联动：行动 → 洞察 → 证据（或洞察 → 证据）
- 支持按观测维度筛选，快速定位某一类问题的证据链

## 6. 典型使用场景

### 场景 A：潜客从 0 到 1

1) 需求挖掘与陪练：完成问题澄清与会前准备  
2) 需求验证与洞察：把“客户话术”翻译成“可验证洞察”  
3) 价值主张创造：产出多个可选方案并勾选  
4) 提案剧本：生成可对外讲述的完整剧本并导出  

### 场景 B：KA 从供货合作到战略合作

1) KA 每周舆情监测：导入周报并长期回看对比  
2) KA 关键动作预测：形成“本周动作与机会”的最新快照  
3) 7C 事实与洞察库：支持多维度多阶推理，实现推理链条与置信度评估可追溯
4) KA 战略与业务提案：基于高价值、高置信度机会精准提案
